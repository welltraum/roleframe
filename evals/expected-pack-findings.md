# Expected findings for the sample pack review

Use this baseline for `review pack evals/files/sample-pack`.

The review should usually identify these categories:

- route exists only in prose and lacks a typed payload
- proof surfaces are present but incomplete
- rollout and visibility are described, but preparedness is still partial
- manifest claims explicit-only invocation, but the route contract is weak
- dashboard must stay derived, while manifest and proof surfaces stay canonical
