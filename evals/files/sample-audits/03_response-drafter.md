# Аудит: агент `response-drafter`

## Вердикт

Итоговый балл: **16/30**

Response-drafter отделен как функция, но prompt поощряет достраивание пробелов "из контекста", что создает риск галлюцинации и ломает честный failure mode.

## Критерии зрелости

| Критерий | Балл | Комментарий |
|---|---:|---|
| Граница функции | 2 | Черновик ответа выделен, но consumer не описан |
| Input | 2 | Вход ожидается от других агентов, но без схемы |
| Control | 1 | Ограничения слабые |
| Mechanism | 2 | Toolkit минимален и соответствует задаче |
| Context engineering | 1 | Нет правил по источникам правды |
| Runtime loop | 1 | Лимиты шагов не заданы |
| Evaluation | 2 | Fixture есть |
| Observability | 1 | Нет logs |
| Safety | 1 | Prompt допускает домысливание |
| Change management | 3 | Артефакты в версии и регрессия планируется |

## Топ-дефициты

1. Prompt разрешает заполнять пробелы догадками.
2. Нет строгого выходного шаблона.
3. Нет contract matrix для handoff от supervisor.
